## 取り組みとスタンス

- **課題ドリブン志向**：ユーザーへの価値提供を起点に要件を洗い出し、中長期で運用しやすいアーキテクチャへ落とし込むことを重視。
- **リアルタイム体験の追求**：Upstash Redis Pub/Sub を用いた SSE 実装や Push 通知で、入札・通知体験の即時性と信頼性を両立。
- **データドリブンな改善**：Redash や SQL を活用した分析で業務改善を推進。仕様検討と効果検証をセットで回すのが得意。
- **継続学習**：Zenn / note で学びを言語化し、アウトプットを通じてナレッジ共有を継続。

## 代表的なプロジェクト

- **無料主義アプリ（freeism-app）**
  - オークションサービスの新規開発から運用まで担当。入札ロジックの楽観的ロック、SSE によるリアルタイム更新、Web Push 通知を実装。
  - UX 改善のため URL とフィルター状態を同期し、検索性と再現性を確保。N+1 解消やインデックス設計でパフォーマンスも最適化。
- **freeism-calc-contrib**
  - OSS やチーム開発での貢献度を定量評価するための Next.js アプリ。GraphQL / GitHub API を活用し、行動の可視化を支援。
- **freeism-depchecker**
  - package.json の依存関係分析を自動化。バージョン差分やライセンスの洗い出しを効率化し、継続的改善を支援。
- **Google Apps Script 自動化**
  - 業務効率化のための各種スクリプトを多数開発。社内運用の改善や KPI モニタリングに活用。
